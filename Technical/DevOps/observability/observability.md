# Observability
## Definitions
- The ability to understand and measure the state of a system based upon data generated by the system.
- To answer the question **"Why is this happening?"**.
## Pillars
### Metrics
- Metrics show the health sign of the system, which are the numeric measurements that show how the system is performing, ie:
	- Latency - Response time
	- Error rate
	- Throughput
	- CPU utilization
	- Memory utilization
	- Request rate
	- Availability
	- Saturation
	- Database query time
	- Network traffic
### Logs
- A log is a captured detailed messages or events of the service or process.
### Traces
- Connect requests across services, showing timing, dependencies, and exceptions.
### Baggage
- Extra information you can attach to a request so it can travel across all services.
### Profiling
- Profiling means looking inside the code itself while it runs to see exactly how much resources each line is using.
## Monitoring and observability
### Monitoring
- Tracks metrics and events.
- Show visibility into symptoms.
- Predefine metrics and events
- Answers "what".
### Observability
- Analyzes outputs (logs, traces, metrics, profiles, ...) to understand system's state.
- Provide context to diagnose root causes.
- Flexible, allow new questions without predefining.
- Answers "why".
